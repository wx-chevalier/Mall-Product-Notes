## 什么是业务风控

业务风控是为了防范业务中产生的各类风险，主要针对以下欺诈行为：

- 薅羊毛
- 刷单炒信
- 数据爬取
- 账户盗用
- 信息冒用
- 盗卡盗刷
- 交易欺诈
- 虚假申贷

## 风控发展历程

### 2013-2014：风控需求兴起

- 2013 年：移动支付普及，O2O 服务兴起
  - 企业大量发放推广红包、优惠券
  - 出现有组织的"薅羊毛"行为
- 2014 年：互联网金融爆发
  - 无门槛优惠吸引大量参与者
  - 欺诈行为开始职业化、团伙化
  - "黑灰产"产业链形成（信息倒卖 → 工具制作 → 攻击实施 → 商品转售）

### 2017 年至今：风控形势严峻

- 《网络安全法》实施
- 企业数字化转型加速
- 业务欺诈成为黑灰产主要手段
- 行业规模：
  - 从业人员超 150 万
  - 市场规模达千亿

## 风控技术演进

### 1.0 策略防护时代

主要特点：基于规则的防护

- 业务规则：如注册 48 小时后才可使用服务
- 名单规则：风险 IP、恶意手机号等黑名单
- 行为规则：监控用户全流程行为

存在问题：

- 规则之间协同性差
- 人为设置局限性大
- 应对风险变化不够灵活

### 2.0 数据模型时代

核心特征：

- 基于大规模数据分析
- 可预测未知威胁
- 形成体系化策略

实施难点：

- 建设周期长（数月到半年）
- 需要专业团队
- 成本投入大

### 3.0 风控中台时代

创新特点：

- 策略、系统、模型的标准化整合
- 数据共享与服务统一
- 打破系统间壁垒
- 提高服务复用率

主要优势：

- 快速构建专属安全系统
- 灵活配置和优化策略
- 有效防范已知和未知风险
- 显著降低建设成本
- 适应业务快速变化
