# 电商系统运营指标体系

## 一、全局运营指标（增强版）

### （一）基础运营指标

1. **总销售额（GMV）**
   - **含义**：指一定时期内销售商品的总金额，反映电商平台的整体交易规模。
   - **计算方式**：GMV = 所有订单的商品销售价格总和。
   - **数据来源**：订单系统
2. **订单总数**
   - **含义**：统计时间段内成功下单的订单数量，体现用户的购买活跃度。
   - **计算方式**：直接统计订单表中状态为已下单的订单记录数。
   - **数据来源**：订单系统
3. **客单价**
   - **含义**：平均每个客户的购买金额，反映客户的消费能力和购买偏好。
   - **计算方式**：客单价 = 总销售额 / 订单总数。
   - **数据来源**：订单系统
4. **转化率**
   - **含义**：潜在客户转化为实际购买客户的比例，衡量营销效果和用户体验。
   - **计算方式**：转化率 = 下单用户数 / 访客数 \* 100%。
   - **数据来源**：订单系统、网站或 APP 的流量统计系统

### （二）运营效率指标

1. **退款率**
   - **含义**：退款订单数占总订单数的比例，反映交易质量。
   - **计算方式**：退款率 =（退款订单数 / 总订单数）× 100%
   - **数据来源**：订单系统记录的退款订单数据和总订单数据
2. **订单取消率**
   - **含义**：取消订单数占总订单数的比例，可预警流程问题。
   - **计算方式**：订单取消率 =（取消订单数 / 总订单数）× 100%
   - **数据来源**：订单系统中的取消订单记录和总订单数据
3. **人均访问深度**
   - **含义**：用户平均浏览页面数，衡量内容对用户的吸引力。
   - **计算方式**：人均访问深度 = 页面浏览总数 / 访问用户数
   - **数据来源**：网站或 APP 的访问日志记录
4. **页面停留时长**
   - **含义**：用户在关键页面的平均停留时间，是反映用户兴趣的指标。
   - **计算方式**：页面停留时长 = 所有用户在关键页面停留时间总和 / 访问关键页面的用户数
   - **数据来源**：网站或 APP 的埋点数据

### （三）渠道效能指标

1. **渠道 ROI（投资回报率）**
   - **含义**：各获客渠道产生的 GMV 与该渠道投入成本的比率。
   - **计算方式**：渠道 ROI = 各获客渠道 GMV / 渠道投入成本
   - **数据来源**：GMV 数据来自订单系统，渠道投入成本来自财务系统
2. **跨渠道转化率**
   - **含义**：通过全渠道转化漏斗（曝光 → 点击 → 加购 → 支付）分析不同渠道在各环节的转化情况。
   - **计算方式**：
     - 曝光 - 点击转化率 =（点击量 / 曝光量）× 100%
     - 点击 - 加购转化率 =（加购量 / 点击量）× 100%
     - 加购 - 支付转化率 =（支付订单数 / 加购量）× 100%
   - **数据来源**：各渠道的推广数据、订单系统和购物车系统数据

## 二、商品分析指标（深度扩展）

### （一）基础商品指标

1. **商品销量**
   - **含义**：特定商品在一定时期内的销售数量，反映商品的受欢迎程度。
   - **计算方式**：统计该商品所有订单中的销售数量总和。
   - **数据来源**：订单系统
2. **商品销售额**
   - **含义**：特定商品的销售金额，体现商品的市场价值。
   - **计算方式**：商品销售额 = 商品销量 \* 商品单价。
   - **数据来源**：订单系统
3. **库存周转率**
   - **含义**：反映库存周转速度，衡量库存管理效率。
   - **计算方式**：库存周转率 = 销售成本 / 平均库存余额。其中，平均库存余额 =（期初库存余额 + 期末库存余额）/ 2。
   - **数据来源**：财务系统的销售成本数据、库存管理系统的库存余额数据
4. **商品动销率**
   - **含义**：有销售记录的商品占总商品数的比例，反映商品的销售活跃程度。
   - **计算方式**：商品动销率 = 有销售的商品数量 / 总商品数量 \* 100%。
   - **数据来源**：商品销售记录和商品库存系统

### （二）商品健康度

1. **滞销商品占比**
   - **含义**：30 天内无销售的商品数量占总商品数的比例。
   - **计算方式**：滞销商品占比 =（30 天无销售商品数 / 总商品数）× 100%
   - **数据来源**：商品销售记录和商品库存系统
2. **商品退货率**
   - **含义**：单品的退货量占其总销量的比例，用于监控商品质量。
   - **计算方式**：商品退货率 =（单品退货量 / 单品总销量）× 100%
   - **数据来源**：订单系统的退货记录和销售记录
3. **商品评价分**
   - **含义**：用户对商品评分的加权平均值，需结合文本分析综合评估商品口碑。
   - **计算方式**：商品评价分 = ∑（各评分等级 × 该等级评价数量）/ 总评价数量
   - **数据来源**：商品评价系统

### （三）商品关联指标

1. **购物车关联率**
   - **含义**：常被同时加购的商品组合数量，反映商品之间的关联性。
   - **计算方式**：统计购物车中同时出现的商品组合数
   - **数据来源**：购物车系统数据
2. **爆品连带率**
   - **含义**：购买爆款商品时搭配其他商品的比例。
   - **计算方式**：爆品连带率 =（购买爆款商品并搭配其他商品的订单数 / 购买爆款商品的订单数）× 100%
   - **数据来源**：订单系统数据

## 三、用户分析指标（分层细化）

### （一）基础用户指标

1. **新增用户数**
   - **含义**：在统计时间段内新注册的用户数量，反映平台的用户增长情况。
   - **计算方式**：统计用户表中注册时间在该时间段内的用户记录数。
   - **数据来源**：用户注册系统
2. **活跃用户数**
   - **含义**：在一定时期内有活跃行为（如登录、浏览、下单等）的用户数量，体现平台的用户粘性。
   - **计算方式**：根据业务定义的活跃行为，统计相应的用户记录数。
   - **数据来源**：网站或 APP 的登录、浏览、下单等行为记录系统
3. **用户留存率**
   - **含义**：反映用户在一段时间后仍然继续使用平台的比例，衡量平台对用户的吸引力。
   - **计算方式**：次日留存率 = 次日活跃用户数 / 当日新增用户数 _ 100%；七日留存率 = 第七日活跃用户数 / 当日新增用户数 _ 100%。
   - **数据来源**：用户注册系统、用户活跃行为记录系统
4. **复购率**
   - **含义**：在一定时期内再次购买的用户占购买用户总数的比例，体现用户对平台的忠诚度。
   - **计算方式**：复购率 = 有两次及以上购买行为的用户数 / 购买用户总数 \* 100%。
   - **数据来源**：订单系统

### （二）用户价值体系

1. **LTV（用户生命周期价值）**
   - **含义**：用户在整个生命周期内为企业贡献的累计利润。
   - **计算方式**：LTV = 用户历史累计消费金额 - 用户历史累计成本
   - **数据来源**：订单系统的消费数据和财务系统的成本数据
2. **RFM 分层**
   - **含义**：根据用户最近消费时间（Recency）、消费频率（Frequency）和消费金额（Monetary）三个维度对用户进行分层。
   - **计算方式**：
     - 最近消费时间：统计用户最后一次消费距离当前日期的天数
     - 消费频率：一定时间内用户的消费次数
     - 消费金额：一定时间内用户的消费总金额
   - **数据来源**：订单系统数据
3. **用户分层占比**
   - **含义**：高价值用户、普通用户、沉睡用户在总用户中的分布比例。
   - **计算方式**：
     - 高价值用户占比 =（高价值用户数 / 总用户数）× 100%
     - 普通用户占比 =（普通用户数 / 总用户数）× 100%
     - 沉睡用户占比 =（沉睡用户数 / 总用户数）× 100%
   - **数据来源**：用户分层结果和用户数据库

### （三）行为分析

1. **用户流失预警**
   - **含义**：对连续 N 天未登录用户的特征进行分析，提前发现可能流失的用户。
   - **计算方式**：筛选出连续 N 天未登录的用户，分析其历史行为、消费习惯等特征
   - **数据来源**：用户登录记录和用户行为分析系统
2. **行为路径分析**
   - **含义**：分析用户从进入平台到浏览、加购、支付的全流程转化情况。
   - **计算方式**：通过埋点记录用户在各个页面和环节的行为，统计不同路径的用户数量和转化率
   - **数据来源**：网站或 APP 的埋点数据
3. **搜索热词分析**
   - **含义**：统计 TOP20 搜索词及其对应的转化率，了解用户搜索需求和购买意向。
   - **计算方式**：搜索词转化率 =（搜索该词后产生购买的用户数 / 搜索该词的用户数）× 100%
   - **数据来源**：搜索系统和订单系统数据

## 四、营销分析指标（效果追踪）

### （一）基础营销指标

1. **广告点击率（CTR）**
   - **含义**：广告被点击的次数与展示次数的比例，衡量广告的吸引力。
   - **计算方式**：CTR = 广告点击次数 / 广告展示次数 \* 100%。
   - **数据来源**：广告投放系统
2. **广告转化率（CVR）**
   - **含义**：通过广告带来的转化行为（如下单、注册等）数量与广告点击次数的比例，评估广告的营销效果。
   - **计算方式**：CVR = 广告带来的转化行为数量 / 广告点击次数 \* 100%。
   - **数据来源**：广告投放系统、订单系统、用户注册系统
3. **营销活动参与率**
   - **含义**：参与营销活动的用户数占活动曝光用户数的比例，反映活动的吸引力和推广效果。
   - **计算方式**：营销活动参与率 = 参与活动的用户数 / 活动曝光用户数 \* 100%。
   - **数据来源**：营销活动系统、用户行为记录系统

### （二）营销效能

1. **优惠券核销率**
   - **含义**：已使用优惠券数量占发放数量的比例，衡量优惠券的使用效果。
   - **计算方式**：优惠券核销率 =（已使用优惠券数量 / 发放优惠券数量）× 100%
   - **数据来源**：优惠券系统的发放和使用记录
2. **满减活动杠杆率**
   - **含义**：满减活动带来的额外 GMV 提升比例。
   - **计算方式**：满减活动杠杆率 = [（活动期间 GMV - 非活动期间平均 GMV）/ 非活动期间平均 GMV] × 100%
   - **数据来源**：订单系统的 GMV 数据
3. **秒杀活动峰值承压**
   - **含义**：秒杀活动期间每秒最高并发订单数，反映系统在活动高峰时的承载能力。
   - **计算方式**：统计秒杀活动期间每秒的订单提交数量，取最大值
   - **数据来源**：订单系统的实时订单记录

### （三）社交媒体指标

1. **UGC 内容传播度**
   - **含义**：用户生成内容（UGC）的分享次数与曝光量的比例，衡量 UGC 的传播效果。
   - **计算方式**：UGC 内容传播度 =（UGC 分享次数 / UGC 曝光量）× 100%
   - **数据来源**：社交媒体平台的 UGC 数据
2. **KOL 带货转化率**
   - **含义**：达人（KOL）推荐商品从点击到购买的转化率。
   - **计算方式**：KOL 带货转化率 =（KOL 推荐商品购买订单数 / KOL 推荐商品点击量）× 100%
   - **数据来源**：社交媒体平台的推广数据和订单系统数据

## 五、财务健康指标（增强监控）

### （一）基础财务指标

1. **毛利润**
   - **含义**：销售收入减去销售成本后的利润，反映企业的盈利基础。
   - **计算方式**：毛利润 = 总销售额 - 商品总成本。
   - **数据来源**：订单系统的销售额数据、财务系统的商品成本数据
2. **净利润**
   - **含义**：扣除所有成本和费用后的利润，体现企业的最终盈利水平。
   - **计算方式**：净利润 = 毛利润 - 运营成本 - 营销费用 - 税收等。
   - **数据来源**：财务系统
3. **毛利率**
   - **含义**：毛利润占总销售额的比例，反映商品的盈利空间。
   - **计算方式**：毛利率 = 毛利润 / 总销售额 \* 100%。
   - **数据来源**：订单系统的销售额数据、财务系统的毛利润数据

### （二）资金效率

1. **应付账款周转天数**
   - **含义**：企业平均支付应付账款的周期，反映与供应商的关系和资金使用效率。
   - **计算方式**：应付账款周转天数 = 365 /（主营业务成本 / 平均应付账款余额）
   - **数据来源**：财务报表中的主营业务成本和应付账款数据
2. **现金流健康度**
   - **含义**：经营活动现金流净额与净利润的比率，评估企业盈利质量和现金流状况。
   - **计算方式**：现金流健康度 = 经营活动现金流净额 / 净利润
   - **数据来源**：财务报表中的现金流和利润数据

### （三）成本结构

1. **单均履约成本**
   - **含义**：平均每个订单的物流、仓储和人工成本总和。
   - **计算方式**：单均履约成本 =（物流成本 + 仓储成本 + 人工成本）/ 订单数
   - **数据来源**：财务系统的成本数据和订单系统的订单数量
2. **退货损失率**
   - **含义**：退货产生的成本损失占 GMV 的比例，反映退货对企业盈利的影响。
   - **计算方式**：退货损失率 =（退货成本损失 / GMV）× 100%
   - **数据来源**：财务系统的退货成本数据和订单系统的 GMV 数据

## 六、供应链指标（智能预警）

### （一）基础供应链指标

1. **订单履行率**
   - **含义**：按时完成交付的订单数量占总订单数量的比例，衡量供应链的交付能力。
   - **计算方式**：订单履行率 = 按时交付的订单数 / 总订单数 \* 100%。
   - **数据来源**：订单系统、物流系统
2. **平均交付时间**
   - **含义**：从订单生成到商品交付给客户的平均时间，反映供应链的效率。
   - **计算方式**：平均交付时间 = 所有订单的交付时间总和 / 订单总数。
   - **数据来源**：订单系统、物流系统

### （二）仓储效能

1. **库龄分析**
   - **含义**：统计不同入库时长（如 30 天、60 天、90 天以上）的库存占总库存的比例，评估库存管理效率。
   - **计算方式**：分别计算各入库时长区间的库存数量占总库存数量的比例
   - **数据来源**：仓储管理系统的库存数据
2. **智能补货预警**
   - **含义**：基于销售预测自动给出补货建议，避免库存短缺或积压。
   - **计算方式**：根据历史销售数据、销售趋势和安全库存水平等因素，运用算法模型预测补货时间和数量
   - **数据来源**：销售系统的历史销售数据和库存管理系统的库存数据

### （三）物流监控

1. **签收异常率**
   - **含义**：因配送问题导致的异常签收订单数占总签收订单数的比例。
   - **计算方式**：签收异常率 =（异常签收订单数 / 总签收订单数）× 100%
   - **数据来源**：物流系统的签收记录
2. **最后一公里时效**
   - **含义**：分区域展示商品配送
